Documentation Technique¶

Introduction¶

Cette section explique l’architecture technique de l’application.

Architecture du Système¶

Le système repose sur une architecture client-serveur avec un serveur basé sur Flask pour gérer les requêtes API et une base de données PostgreSQL pour le stockage des données.

Composants principaux¶

  1. Serveur : Gère les requêtes HTTP, le routage, et la logique métier.

  2. Base de données : Utilise PostgreSQL pour stocker les informations de l’application.

  3. API : Les endpoints API sont définis pour interagir avec les données.
